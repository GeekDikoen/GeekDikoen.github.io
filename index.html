hexo搭建与上传GitHub

关于使用hexo搭建博客以及上传至GitHub的方法

前言

过程比较复杂（对于不怎么懂的人来说），文章比较长，请耐心看 ，此教程主要适用于第一次搭建、不怎么懂的用户。

1、下载git与node。

这两者对hexo的搭建起着必要性作用，所以一定要先下载到电脑。对于Windows、Mac和Linux都适用，链接奉上[git](Git (git-scm.com))、[node](Node.js (nodejs.org))，点击即可跳转。

2、安装hexo

在下载好git与node之后，在桌面上单击鼠标右键，可出现Git Bush Here的选项，如图

点击，键入npm install hexo-cli -g,完成后，单击enter键，即可安装hexo

3、搭建本地Hexo

随便在一个硬盘里新建一个文件夹，文件夹的名字可以是“blog”。这个文件夹用来存放你博客的内容。

在新建的文件夹内，打开Git Bush Here，如上。输入hexo init，回车。成功后会发现文件夹内多了好多新文件。接下来执行npm install，现在本地博客已经搭建完成了。如果想查看，可以执行hexo s,会打开服务器，接下来根据刚刚INFO的反馈，打开网址。一般都是localhost:4000这个网址。

4、注册GitHub以及创建储存库

既然想上传至GitHub，那我们先要拥有一个账号。注册流程就不说了，这个真的没什么可以说的。

注册号账号后，就来新建一个储存库吧。在主页面的右上角，有个小加号，选择第一个，点击，如图

仓库的名字就是用户名+github.io，一定要是这个格式的，不然识别不了

Note: 用户名即为GitHub的用户名，不要乱取名字！！！

可见方式选择in public。

5、配置SSH key

这是将博客上传到GitHub的一个关键操作。其实就是一个密钥，公钥在GitHub上，私钥在你的电脑里，两者相互匹配，才能成功将你的文章发布于互联网。

执行以下代码

成功后，会出现这个图像（差不多就行，不需要一模一样）

中途需要按两次enter键，具体什么时候从它弹出的信息中决定

这时，你的ssh key已经储存在电脑里了，一般情况下在C盘的用户文件夹里，文件夹的名称为.ssh，里面有个文件，其名为id_rsa.pub，用记事本打开，复制里面的内容备用

接下来打开GitHub，还是右上角，不过不完全是，如图

点开后，在左侧菜单栏选择SSH and GPG keys，然后点击`New SSH key

6、同步至GitHub

我们需要安装一个插件来同步，执行npm install hexo-deployer-git --save，回车以安装。

在你存放博客的目录里，有一个文件叫做_config.yml，用记事本打开就可以。

找到此文件中的deploy字段，按照以下配置deploy

1 type: ‘git’ 冒号后面有空格!!!

2 repo: git@github.com:XX/XX.github.io.git 只是示意,repo从仓库里能看得到,一会放图

3 branch: master

一定不要忘记冒号后面有空格!!!!!

修改好后保存

开始上传

执行hexo d,等待一会,内容就自动发布到GitHub上了

7、查看你发布的文章

打开GitHub→仓库→库里面的settings→下划左侧有一选项为pages→点击→就能看到了

现在你的文章已经发布到这个链接了,默认里面只有一篇名为Hello world的文章
